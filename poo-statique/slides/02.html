POO : statique

On peut définir des propriétés ou méthodes statiques.

Elles appartiennent alors à la classe, et non pas à l'instance de classe, et peuvent être utilisées sans avoir besoin d'instancier la classe :

class Date {
    public function today() {
        return date('d/m/Y');
    }
    public function tomorow() {
        $datetime = new DateTime('tomorrow');
        return $datetime->format('d/m/Y');
    }
}

$date = new Date(); // ??? mais pourquoi
echo 'aujourd\'hui : ' . $date->today() . PHP_EOL;
echo 'demain : ' . $date->tomorow() . PHP_EOL;
class Date {
    public static function today() {
        return date('d/m/Y');
    }
    public static function tomorow() {
        $datetime = new DateTime('tomorrow');
        return $datetime->format('d/m/Y');
    }
}

echo 'aujourd\'hui : ' . Date::today() . PHP_EOL;
echo 'demain : ' . Date::tomorow() . PHP_EOL;

Le mot-clé static permet d'indiquer que la propriété ou la méthode est statique.

On utilise l'opérateur :: (Paamayim Nekudotayim) pour accéder aux éléments statiques.

$this n'est pas disponible dans une méthode statique.
