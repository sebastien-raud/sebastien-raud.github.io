API : historique

  • Concept initié dans les années 1940
    • bibliothèque logicielle modulaire pour l'EDSAC
    • sous forme de cartes perforées organisées dans des classeurs
    • catalogue de bibliothèque (documentation sur chaque sous-programme) : indique comment utiliser / appeler les sous-programmes : API
  • 1968 : terme "application program interface" utilisé dans l'article Data structures and techniques for remote computer graphics
  • 1974 : terme "API" dans l'article The Relational and Network Approaches: Comparison of the Application Programming Interface.
    Utilisé pour définir une séparation entre la gestion des données (base de données) et le reste de l'application.
  • Années 80-90 : beaucoup de recherches et d'innovations
    • protocole RPC pour faire des appels de procédures distants
    • API POSIX
    • API Windows Win16 puis Win32 : accès au système de fichiers, aux périphériques, bibliothèques d'interface graphique...
    • API d'applications (ex: ajourd'hui le DOM est une API des navigateurs pour permettre à Javascript de manipuler HTML)
    • CORBA : services séparés accessibles à distance
  • 1998 : SOAP
    • échanges par HTTP (le plus souvent)
    • utilisation de XML pour les messages
    • ouvert, indépendant de la plate-forme et du langage
    • lourd et compliqué à mettre en œuvre
  • 2000 : définition de l'architecture REST par Roy Fielding
