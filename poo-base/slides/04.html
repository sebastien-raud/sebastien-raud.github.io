POO

Pourquoi la programmation orientée objet ?

La POO est introduite avec le langage Simula 67 en 1967 puis par le langage Smalltalk en 1972.

Ce type de programmation prend son essort en 1983 avec le langage C++.

La POO répond à différents problèmes :

  • organiser différement le code
  • en simplifier la maintenance, les évolutions
  • faciliter la réutilisation du code
  • masquer certains niveaux de complexité du code

Mais la POO a aussi ses inconvénients :

  • l'approche objet est moins intuitive que l'approche par fonctions
  • elle est moins naturelle pour l'esprit humain
  • le développeur doit concevoir et modéliser avant de programmer
