Du MCD au MLD (2)

Relation 1-1 : spécificités

Le champ qui contient la clé étrangère doit être déclaré UNIQUE :

  • si on a un champ prefecture.depatement_id, on ne veut que une valeur de departement_id dans tous les enregistrements, qui correspond au seul département d'une seule préfecture
  • si on a un champ departement.prefecture_id, on ne veut que une valeur de prefecture_id dans tous les enregistrements, qui correspond à la seule préfecture d'un seul département

On peut définir le dictionnaire des données suivant :

Table prefecture

Champ Type spécificités Description
id INT PRIMARY KEY, NOT NULL, AUTO_INCREMENT, UNSIGNED Identifiant unique d'un enregistrement
ville VARCHAR(64) NOT NULL Ville dans laquelle est installée la préfecture
prefet VARCHAR(128) NOT NULL Nom et prénom du préfet
created_at TIMESTAMP NOT NULL, DEFAULT CURRENT_TIMESTAMP Date de création de l'enregistrement
updated_at TIMESTAMP NULL Date de dernière modification de l'enregistrement
departement_id INT NOT NULL, UNIQUE, UNSIGNED, FOREIGN KEY(departement.id) Identifiant du département

Table departement

Champ Type spécificités Description
id INT PRIMARY KEY, NOT NULL, AUTO_INCREMENT, UNSIGNED Identifiant unique d'un enregistrement
nom VARCHAR(16) NOT NULL Nom du département
created_at TIMESTAMP NOT NULL, DEFAULT CURRENT_TIMESTAMP Date de création de l'enregistrement
updated_at TIMESTAMP NULL Date de dernière modification de l'enregistrement
