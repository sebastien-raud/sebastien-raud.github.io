API REST / RESTful

  • Basé sur HTTP
  • Une URL = une ressource
  • Un verbe HTTP = une opération
    • GET : retourner la ressource
    • POST : créer la ressource
    • PATCH : mettre à jour partiellement la ressource
    • PUT : mettre à jour complètement la ressource
    • DELETE : supprimer la ressource
  • Réponse HTTP :
    • représentation de la ressource (HTML, XML, JSON, CSV...)
    • code HTTP : réponse sur le succès ou non de l'opération (200, 201, 404...)
