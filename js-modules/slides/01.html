Pour organiser encore plus son code !

Mais on peut organiser plus !

Mais pourquoi organiser plus ?

Pour avoir du code plus lisible.

Pour avoir du code bien "découpé".

Pour réduire les risques d'introduction de bugs.

Pour faciliter la maintenance et l'évolutivité du code.

On organise déjà nos projets

On sépare le HTML, le CSS et le JS.

En PHP on sépare les données des templates pour la génération du code HTML.

En JS on place les fonctionnalités dans des fichiers dédiés.

Mais est-ce suffisant ?

Fichier simple-counter.js

let counter = 0;
document.getElementById('button').addEventListener('click', handleCounter);
function handleCounter() {
    counter++;
    document.getElementById('counter').textContent = counter;
}

Tout est dans le contexte / scope global.

Est-ce qu'un fichier other.js ne risque pas de modifier ma variable counter ?

Est-ce que je ne risque pas d'autres effets indésirables ?
